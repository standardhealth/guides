Primary local value sets used in this Implementation Guide

Name Definition
CarePlanChangeReasonVS

Reasons that a care plan was changed. Similar to TreatmentTerminationReasonVS

Primary external value sets used in this Implementation Guide

Name Definition
Activity Reason

Example codes indicating the reason for a clinical activity being performed.
All Security Labels

A single value set for all security labels defined by FHIR.
Care Plan Activity

Example Codes for types of activities that can appear in a care plan.
Care Plan Category

Example codes indicating the category a care plan falls within. Note that these are in no way complete and may not even be appropriate for some uses.
CarePlanActivityCategory

High-level categorization of the type of activity in a care plan.
CarePlanActivityStatus

Indicates where the activity is at in its overall life cycle.
CarePlanRelationship

Codes identifying the types of relationships between two plans.
CarePlanStatus

Indicates whether the plan is currently being acted upon, represents future intentions or is now a historical record.
GoalStatusReason

Example codes indicating the reason for a current status. Note that these are in no way complete and may not even be appropriate for some uses.
Participant Roles

Roles of Participants that may be included in a CarePlan.Participants, or in an EpisodeOfCare.CareTeam. Defined as: Is a Person, Healthcare professional (occupation) or Healthcare related organization (qualifier value).
SNOMED CT Medication Codes

This value set includes all Medication codes from SNOMED CT - provided as an exemplar.
